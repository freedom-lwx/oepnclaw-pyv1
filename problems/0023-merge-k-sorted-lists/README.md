# 23. 合并K个升序链表

**难度:** Hard

**链接:** https://leetcode.com/problems/merge-k-sorted-lists/

## 题目描述

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

## 示例

```
输入: lists = [[1,4,5],[1,3,4],[2,6]]
输出: [1,1,2,3,4,4,5,6]

解释: 
1->4->5,
1->3->4,
2->6
合并后:
1->1->2->3->4->4->5->6
```

## 思路

这道题是「合并两个有序链表」的升级版。

### 方法一：最小堆（优先队列）

核心思想：每次从所有链表的头节点中选最小的那个。

1. 把所有链表的头节点放入最小堆
2. 弹出最小节点，加入结果链表
3. 把该节点的 next 加入堆（如果有）
4. 重复直到堆为空

### 方法二：分治合并

类似归并排序：

1. 两两合并链表
2. 递归/迭代直到只剩一个

## 复杂度

| 方法 | 时间 | 空间 |
|------|------|------|
| 最小堆 | O(N log k) | O(k) |
| 分治 | O(N log k) | O(log k) |

- N = 所有节点总数
- k = 链表数量

## 关键点

1. **堆中存储 (val, index, node)**：避免直接比较 ListNode
2. **虚拟头节点**：简化边界处理
3. **分治 vs 堆**：分治空间更优，堆代码更直观
